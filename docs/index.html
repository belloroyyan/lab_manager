Quick Start — Admin Console

The admin console (main.py) is what you run on your own workstation to operate the dashboard and orchestrate actions across the lab. It's Windows-first and depends on a handful of packages.

  1. Install Python 3.11+ and dependencies. pip install colorama psutil requests pyzipper fpdf winshell pyyaml cryptography
  2. Run the console. python main.py — first run auto-generates settings.json and creates logs/, reports/, and venvs/ next to the project.
  3. Pick a menu option. Numbers/letters route to the relevant sub-menu — Network, Backup, Venv, Cleanup, I/O, Logs, and Settings all prompt for whatever input they need (paths, IPs, package names).
  4. Deploy agents to lab PCs. See Listener / Agent Installation below for turning a lab PC into a manageable endpoint.

Security Model

Every message between the admin console and a lab agent is encrypted — nothing about remote control relies on network obscurity alone.

Per-Machine Encryption Key

A Fernet key lives inside a hidden identity file on each machine, generated once and exported as secret.key for manual transfer (USB, not email) to agents. It is never stored in settings.json.

Password-Gated Setup

First-run listener setup requires creating a password before an identity file exists. It's stored only as a salted PBKDF2-HMAC-SHA256 hash — never in plaintext.

Time-Boxed Remote Commands

SHUTDOWN, RESTART, and LOGOUT are rejected unless ENABLE_REMOTE was sent first. The window auto-expires after up to 60 minutes.

Residual risk: anyone holding the shared key can command that agent — there's no per-operator identity or command signing yet. Treat secret.key like a credential and keep ENABLE_REMOTE windows short.

Listener / Agent Installation

Copy the agent build below onto every lab PC that should show up in the admin console. Each agent runs a one-time password-protected setup, then listens quietly on UDP for encrypted requests.

Standalone agent build. Place on lab PCs you want to monitor and control remotely.

Setup Steps

  1. Copy listener.exe to the target PC. Any folder works — the agent creates its own identity file under %PROGRAMDATA%/LabManager the first time it runs.
  2. Run it once and complete first-run setup. You'll be asked to set a local password (protects re-running setup later), and optionally a school/lab name and log file path.
  3. Import the shared secret.key. When prompted, point to the secret.key file generated from the admin console's identity (via generate_and_export_key()) — transfer it by USB, not over email or chat.
  4. Allow the firewall rule. Run as Administrator once so the agent can add its own inbound UDP rule (default port 8088); otherwise allow it manually when prompted.
  5. Leave it running. After first-run setup, the console window hides itself and the agent listens silently until the PC restarts or the process is killed.
